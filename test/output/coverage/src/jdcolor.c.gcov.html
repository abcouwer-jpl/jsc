      36             : /*
      37             :  * YCbCr is defined per Recommendation ITU-R BT.601-7 (03/2011),
      38             :  * previously known as Recommendation CCIR 601-1, except that Cb and Cr
      39             :  * are normalized to the range 0..MAXJSAMPLE rather than -0.5 .. 0.5.
      40             :  * sRGB (standard RGB color space) is defined per IEC 61966-2-1:1999.
      41             :  * sYCC (standard luma-chroma-chroma color space with extended gamut)
      42             :  * is defined per IEC 61966-2-1:1999 Amendment A1:2003 Annex F.
      43             :  * bg-sRGB and bg-sYCC (big gamut standard color spaces)
      44             :  * are defined per IEC 61966-2-1:1999 Amendment A1:2003 Annex G.
      45             :  * Note that the derived conversion coefficients given in some of these
      46             :  * documents are imprecise.  The general conversion equations are
      47             :  *
      48             :  *        R = Y + K * (1 - Kr) * Cr
      49             :  *        G = Y - K * (Kb * (1 - Kb) * Cb + Kr * (1 - Kr) * Cr) / (1 - Kr - Kb)
      50             :  *        B = Y + K * (1 - Kb) * Cb
      51             :  *
      52             :  *        Y = Kr * R + (1 - Kr - Kb) * G + Kb * B
      53             :  *
      54             :  * With Kr = 0.299 and Kb = 0.114 (derived according to SMPTE RP 177-1993
      55             :  * from the 1953 FCC NTSC primaries and CIE Illuminant C), K = 2 for sYCC,
      56             :  * the conversion equations to be implemented are therefore
      57             :  *
      58             :  *        R = Y + 1.402 * Cr
      59             :  *        G = Y - 0.344136286 * Cb - 0.714136286 * Cr
      60             :  *        B = Y + 1.772 * Cb
      61             :  *
      62             :  *        Y = 0.299 * R + 0.587 * G + 0.114 * B
      63             :  *
      64             :  * where Cb and Cr represent the incoming values less CENTERJSAMPLE.
      65             :  * For bg-sYCC, with K = 4, the equations are
      66             :  *
      67             :  *        R = Y + 2.804 * Cr
      68             :  *        G = Y - 0.688272572 * Cb - 1.428272572 * Cr
      69             :  *        B = Y + 3.544 * Cb
      70             :  *
      71             :  * To avoid floating-point arithmetic, we represent the fractional constants
      72             :  * as integers scaled up by 2^16 (about 4 digits precision); we have to divide
      73             :  * the products by 2^16, with appropriate rounding, to get the correct answer.
      74             :  * Notice that Y, being an integral input, does not contribute any fraction
      75             :  * so it need not participate in the rounding.
      76             :  *
      77             :  * For even more speed, we avoid doing any multiplications in the inner loop
      78             :  * by precalculating the constants times Cb and Cr for all possible values.
      79             :  * For 8-bit JSAMPLEs this is very reasonable (only 256 entries per table);
      80             :  * for 9-bit to 12-bit samples it is still acceptable.  It's not very
      81             :  * reasonable for 16-bit samples, but if you want lossless storage you
      82             :  * shouldn't be changing colorspace anyway.

      84             :  * values for the G calculation are left scaled up, since we must add them
