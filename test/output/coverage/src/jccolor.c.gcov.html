      26             : /*
      27             :  * YCbCr is defined per Recommendation ITU-R BT.601-7 (03/2011),
      28             :  * previously known as Recommendation CCIR 601-1, except that Cb and Cr
      29             :  * are normalized to the range 0..MAXJSAMPLE rather than -0.5 .. 0.5.
      30             :  * sRGB (standard RGB color space) is defined per IEC 61966-2-1:1999.
      31             :  * sYCC (standard luma-chroma-chroma color space with extended gamut)
      32             :  * is defined per IEC 61966-2-1:1999 Amendment A1:2003 Annex F.
      33             :  * bg-sRGB and bg-sYCC (big gamut standard color spaces)
      34             :  * are defined per IEC 61966-2-1:1999 Amendment A1:2003 Annex G.
      35             :  * Note that the derived conversion coefficients given in some of these
      36             :  * documents are imprecise.  The general conversion equations are
      37             :  *      Y  = Kr * R + (1 - Kr - Kb) * G + Kb * B
      38             :  *      Cb = 0.5 * (B - Y) / (1 - Kb)
      39             :  *      Cr = 0.5 * (R - Y) / (1 - Kr)
      40             :  * With Kr = 0.299 and Kb = 0.114 (derived according to SMPTE RP 177-1993
      41             :  * from the 1953 FCC NTSC primaries and CIE Illuminant C),
      42             :  * the conversion equations to be implemented are therefore
      43             :  *      Y  =  0.299 * R + 0.587 * G + 0.114 * B
      44             :  *      Cb = -0.168735892 * R - 0.331264108 * G + 0.5 * B + CENTERJSAMPLE
      45             :  *      Cr =  0.5 * R - 0.418687589 * G - 0.081312411 * B + CENTERJSAMPLE
      46             :  * Note: older versions of the IJG code used a zero offset of MAXJSAMPLE/2,
      47             :  * rather than CENTERJSAMPLE, for Cb and Cr.  This gave equal positive and
      48             :  * negative swings for Cb/Cr, but meant that grayscale values (Cb=Cr=0)
      49             :  * were not represented exactly.  Now we sacrifice exact representation of
      50             :  * maximum red and maximum blue in order to get exact grayscales.
      51             :  *
      52             :  * To avoid floating-point arithmetic, we represent the fractional constants
      53             :  * as integers scaled up by 2^16 (about 4 digits precision); we have to divide
      54             :  * the products by 2^16, with appropriate rounding, to get the correct answer.
      55             :  *
      56             :  * For even more speed, we avoid doing any multiplications in the inner loop
      57             :  * by precalculating the constants times R,G,B for all possible values.
      58             :  * For 8-bit JSAMPLEs this is very reasonable (only 256 entries per table);
      59             :  * for 9-bit to 12-bit samples it is still acceptable.  It's not very
      60             :  * reasonable for 16-bit samples, but if you want lossless storage you
      61             :  * shouldn't be changing colorspace anyway.
      62             :  * The CENTERJSAMPLE offsets and the rounding fudge-factor of 0.5 are included
      63             :  * in the tables to save adding them separately in the inner loop.
      64             :  */
