     140             : /*
     141             :  * Several decompression processes need to range-limit values to the range
     142             :  * 0..MAXJSAMPLE; the input value may fall somewhat outside this range
     143             :  * due to noise introduced by quantization, roundoff error, etc.  These
     144             :  * processes are inner loops and need to be as fast as possible.  On most
     145             :  * machines, particularly CPUs with pipelines or instruction prefetch,
     146             :  * a (subscript-check-less) C table lookup
     147             :  *              x = sample_range_limit[x];
     148             :  * is faster than explicit tests
     149             :  *              if (x < 0)  x = 0;
     150             :  *              else if (x > MAXJSAMPLE)  x = MAXJSAMPLE;
     151             :  * These processes all use a common table prepared by the routine below.
     152             :  *
     153             :  * For most steps we can mathematically guarantee that the initial value
     154             :  * of x is within 2*(MAXJSAMPLE+1) of the legal range, so a table running
     155             :  * from -2*(MAXJSAMPLE+1) to 3*MAXJSAMPLE+2 is sufficient.  But for the
     156             :  * initial limiting step (just after the IDCT), a wildly out-of-range value

     158             :  * off the end of memory and getting a bad-pointer trap, we perform the
     159             :  * post-IDCT limiting thus:

     162             :  * samples.  Under normal circumstances this is more than enough range and
     163             :  * a correct output will be generated; with bogus input data the mask will
